Safe Companion — Privacy Policy

Last updated: 2026-04-26

Safe Companion is built to help senior users avoid scams. Privacy is core to that mission. This policy describes exactly what data the app uses, how it is processed, and what we never do.

Plain-English summary: Safe Companion analyses scam-detection signals on your device. The only time message text is sent off the device is when the on-device classifier is uncertain AND you have configured a Claude API key — in which case the text snippet is sent to Anthropic for analysis. No identifiers, contacts, location, or message content are sent to Safe Companion's developers, ever.

What Safe Companion accesses on your device

  • Notifications — to read incoming SMS and email previews so it can warn you about scams. Reads only; never modifies.
  • Contacts — to recognise calls and texts from people you already know.
  • Phone state — to detect when a flagged call begins/ends so a family alert can be sent if it lasts a long time.
  • Microphone — only when you tap the voice button or run the room ultrasonic scan.
  • Camera — only when you take a photo to be analysed or scan a QR code.
  • Location (coarse + fine) — required by Android to read nearby WiFi network names; we do not store or send your location anywhere.
  • Bluetooth scan — only during a Room Scan, to look for devices with suspicious names.
  • Installed apps list — the App Checker scans installed packages to flag risky permissions; the list never leaves your device.
  • Usage access (optional) — used only to detect when a payment app comes to the foreground, so we can show a safety reminder.
  • SMS sending — used only to text the family contacts you have configured, when a high-risk scam pattern is detected. Each message goes to the numbers you entered yourself; we do not send to any other recipient.

Data sent off your device

Anthropic (Claude API) — only with your API key, only for ambiguous cases

If you configure a Claude API key, message snippets that the on-device classifier marks "needs further analysis" are sent to api.anthropic.com for a verdict. We do not include your phone number, contacts, or device identifiers. Anthropic processes the request under their privacy policy and does not retain prompts beyond their session window. If you do not configure a key, nothing is sent.

ElevenLabs — only with your API key, only for voice playback

If you configure an ElevenLabs API key, the agent's reply text is sent to api.elevenlabs.io to synthesise a natural voice. No personal data is included.

Google Cloud Text-to-Speech — only with your API key, fallback voice tier

Same pattern as ElevenLabs. Only the reply text and your key. Optional.

RSS feeds — read-only, anonymous

The Security News feed pulls public RSS articles from AARP, FTC, FBI, BBB, and Snopes. These are anonymous HTTP requests; no identifiers attached.

What Safe Companion never does

  • Sell or share your data with advertisers, data brokers, or analytics SDKs.
  • Upload your contacts, call log, location, or photos.
  • Run trackers, ad networks, or third-party SDKs beyond the optional API services listed above.
  • Record your phone calls or store audio.

On-device storage

Scam alerts, call screening logs, and your settings are stored locally in an encrypted Room database. API keys are encrypted with AES/GCM in the Android Keystore — they never leave your device in plaintext. Backup is disabled, so a stolen device backup will not contain Safe Companion data.

Children

Safe Companion is designed for adults (age 18+) and is not directed at children.

Your rights

You can clear all stored data at any time from Settings → Clear All Data. Removing the app removes all local data. There is no server-side account, so there is nothing to delete on our end.
